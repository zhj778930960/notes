      # Swagger接口文档

      # 什么是swagger?

      • Swagger 是一个规范且完整的框架，用于生成、描述、调用和可视化 RESTful风格的 Web 服务。

        Swagger的目标是对 REST API定义一个标准且和语言无关的接口，可以让人和计算机拥有无须访问源码、文档或网络流量监测就可以发现和理解服务的能力。当通过 Swagger进行正确定义，用户可以理解远程服务并使用最少实现逻辑与远程服务进行交互。与为底层编程所实现的接口类似，Swagger 消除了调用服务时可能会有的猜测。

      # 优势
      • 支持 API自动生成同步的在线文档：使用 Swagger 后可以直接通过代码生成文档，不再需要自己手动编写接口文档了，对程序员来说非常方便，可以节约写文档的时间去学习新技术。
      • 提供 Web 页面在线测试 API：光有文档还不够，Swagger 生成的文档还支持在线测试。参数和格式都定好了，直接在界面上输入参数对应的值即可在线测试接口。

      # 如何在node项目中使用swagger

      # 方案一、需要两个插件 koa2-swagger-ui 和 swagger-jsdoc
      • 1. 安装 koa2-swagger-ui 和  swagger-jsdoc

      • 优势：

        • swagger-jsdoc 可以将swagger以注释的方式写入，接口中，可以使得在程序员在写接口的时候，直接就可以同步写入swagger注释
        • 并且开发人员在调整接口代码的时候，很方便的就可以修改注释， 不至于忘记了维护接口文档。

      • 缺点：

        • 因为代码和接口文档是耦合在一起的，所以，在router的文件中，需要出现大量的注释，不方便查看
        • 有的接口需要很多的参数，那么swagger的注释就需要写的很长，挡在swagger中出现语法错误，缩进错误的时候，无法直接定位到出错点

      • 代码：

        • 首先创立好项目之后，在app.js中加入如下代码

          const Koa = require("koa");
const koaSwagger = require("koa2-swagger-ui");

const app = new Koa();

app.use(koaSwagger({
    routePrefix: "/api-docs",  // swagger的访问地址 http://localhost:port端口号/api-docs  =》                          http://localhost:3000/api-docs
    swaggerOptions: {
        url: "/swagger.json"   // 这个是用于拿到swagger.json的接口数据
    }
}));

/**
 * swagger 路由配置，和一些基本信息的配置  引入swagger路由  这样才能在浏览器访问swagger接口文档界面
 * 配置基本与别的接口没有什么区别，就是增加了一些swagger自己的配置
 * 例如要读取那些接口，生成文档  swagger.json 界面 生成的json数据如何生成
*/
const swagger = require("./utils/swagger.js");
app.use(swagger.routes(), swagger.allowedMethods());

/**
 * 普通的接口配置，这样就能在接口文档和业务中进行访问了
 * 就是这些接口文件中需要加入swagger的注释格式，才能够生成swagger文档
 */
const login = require("./router/login.js");
app.use(login.routes(), login.allowedMethods());


app.listen(3000, () => {
    console.log("启动成功！！")
})

        • 配置swagger文件， 配置swagger.json的路由

          const router= require("koa-router")();
const swaggerJsdoc = require("swagger-jsodc"); // 就是主要生成接口文档，并且能返回对应的json
const path = require("path");
// 基本配置
const swaggerDefinition = {
     info: {
         titile: "Api",   # 一般设置成为项目的名称最好  例如 api-common-fs
         version: "0.0.1", # 版本号
         description: "对于接口文档的描述"
     }
}

// 配置汇总，在上述配置同级别，接入那些接口需要进行文档的生成
const options = {
    swaggerDefinition,  // 将上述的基本信息配置进来
    apis: [path.join(__dirname, "../router/*.js")]   # 将router文件夹下的，所有js文件都导入成接口文档，只要接口上面加入了注释
}


// 配置json 数据 将options 配置中的信息收集到之后，转成json 文件的，当开发人员访问。swagger.json 界面就可以访问到对应的json数据
const swaggerSpec = swaggerJsdoc(options);


router.get("/swagger.json", async () => {
    ctx.set("Content-Type": "application/json");  // 设置返回头
    ctx.body = swaggerSpec;
});

module.exports = router   //  导出swagger 路由 在app.js中导入之后配置好就可以使用swagger.json

        • 配置其余路由（接口）文件和增加注释（接口文档的生成数据）

          const router = require("koa-router")();

# 设置swagger 注释  配置属性   空格缩进比较严格


/**
 * @swagger
 * /login:                      # 接口
 *   post:                      # 请求方式
 *     tags:
 *       - 登陆页面               # 标签
 *     parameters:              # 参数 
 *       - name: name            # 第一个参数，name
 *         in: query
 *         required: true
 *         description: 用户名称
 *         type: string
  *      - name: password            # 第二个参数，password
 *         in: query
 *         required: true
 *         description: 用户密码
 *         type: string
 *     summary: 更在接口后面的副标题，用于说明是哪个接口
 *     description: 登陆的方法
 *     responses:
 *         200:
 *           description: 成功返回
 */
router.post("/login", async () => {
     ctx.body = {
          code: 0,
          msg: "请求成功"，
          errorMsg: null,
          object: "ihj23i4h4n23df9a2n0hnf38naehne3423mjf890234naskhdf839845h"
     }
});


# tags 相同 就会在同一个标签下，这样就可以将同一界面的接口，放置到同一个tags下，这样就调理比较清楚，当然同一个js下基本也就是一个界面的接口

/**
 * @swagger
 * /logout:                      # 接口
 *   get:                      # 请求方式
 *     tags:
 *       - 登陆页面               # 标签
 *     summary: 更在接口后面的副标题，用于说明是哪个接口
 *     description: 登陆的方法
 *     responses:
 *         200:
 *           description: 成功返回
*/
router.get("/logout", async () => {
     ctx.body = {
          code: 0,
          msg: "登出成功"，
          errorMsg: null,
          object: null
     }
});


module.exports = router;
